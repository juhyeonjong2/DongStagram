기업 커뮤니티 사이트

사원이 들어온 경우    ->  로그인 페이지(id, pw, 비밀번호 찾기),(회원가입은 x) -> 나의 블로그로 이동 -> 탭메뉴 등으로 이동 시 ->모든 팀들의 게시판을 작게 보여줌
관리자가 들어온 경우 ->  로그인 페이지 -> 관리자 페이지 -> 1. 사원 등록(이메일, 이름(가입한 순으로 번호매김), 파트/팀 ,[임시등록, 가입요청])
		  			 	                    -> 이메일로 가입페이지 링크 보냄 -> 링크로 들어가면 id, pw를 적어 회원가입
						-> 2.


크게 구현해야 할 리스트 ->
   1. 나의 블로그, 팀의 게시글
   2. 회원가입
   3. 관리자 페이지	
   4. 메세지 기능 (메세지는 당사자끼리만 볼 수 있게, 메세지는 언제든지 작성 가능하게 팝업으로)
   5. 요구사항 분석서와 스토리 보드 작업하면서 누락된 기능 추가


관리자 : 예) 사장님  - > 회원 : 예) 사원

회사 전용 공간(나의 회사) - > a 파트/팀 - > a 사원
			       	 - > b 사원
			       	 - > c 사원

                                - > b 파트/팀 - > a 사원
			       	 - > b 사원
			       	 - > c 사원

                                - > c 파트/팀 - > a 사원
			       	 - > b 사원
			       	 - > c 사원

각 사원은 블로그(리포트)를 작성 할 수 있고 작성한 블로그는 타인이 보고 댓글을 남길 수 있다.

각 팀은 팀끼리만 글을 작성할 수 있다(기본적으로 보는건 전부 가능(자랑 게시판))
